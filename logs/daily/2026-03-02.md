## Work Completed

1. **Contact form — Pipeline selector** — Added pipeline stage pill selector to the add contact form, matching the existing edit modal style. Defaults to "Nieuw". Backend POST handler now accepts and validates `pipeline_stage`.

2. **Contact form — Expertgroep source** — Added "Expertgroep" as a new option in the Bron dropdown on both the add and edit contact forms.

3. **Anomaly cell background — red → grey** — Changed `--trend-anomaly-bg` from `rgba(227,1,1,0.1)` (red) to `rgba(14,50,97,0.08)` (navy-grey). Red implied negative, but 50%+ anomalies can be positive too. Info popover text updated from "Rood gemarkeerd" to "Gemarkeerd".

4. **Trend-colored tooltips** — Hover tooltips on anomaly cells now show red background (`#E30101`) for negative YoY changes and green (`#2C7A3D`) for positive. Pure CSS via `data-trend` attribute — no new components or dependencies.

5. **Social content pipeline — zero-error architecture** — Built complete pipeline from scratch after analysis revealed the previous hand-typed approach had data errors (SVB 2024 off by €243M), broken dedup (key was only `(module, entity)` with suffix hacks like "Trigion2"), and a poisoned registry (589 entries for batches 3-6 that didn't exist as files). New architecture: `Database → extract_facts.py → facts/*.csv → generate_posts.py → posts/*.csv`, with `verify.py` for re-verification. Every number traces back to the database through a verified chain.

6. **Fact extraction (extract_facts.py)** — Connects to Supabase via psycopg2 (reads DATABASE_URL from backend/.env), runs 18 SQL queries across all 6 modules + universal_search, outputs 18 CSV files with 1,921 machine-verified facts. Each fact has a unique ID (e.g., INSTR-TOP-001), query_ref, and verified_at timestamp.

7. **Fact verification (verify.py)** — Re-runs source queries against live DB and compares to stored CSV values. Supports `--module` and `--fact` filters. Result: 950 PASS, 0 CHANGED, 0 MISSING.

8. **Post generation (generate_posts.py)** — Reads facts CSVs and generates posts in 5 formats (scale_shock, comparison, concentration, category_reveal, curiosity). Features: entity blocklist for noise data (AVG restrictie, Niet toegewezen, anonymized entries), Dutch number formatting (€13.100.000.000), staffel bracket language for inkoop/publiek, proper singular/plural grammar, entity name cleanup (capitalize gemeente/provincie/stichting, collapse double spaces). Result: 2,435 posts across 25 batches, 0 issues.

9. **Data quality fixes during pipeline build:**
   - NULL entity in instrumenten_aggregated (€1.3B with no name) → filtered with WHERE clause
   - "(leeg)" entities passed blocklist → added to ENTITY_BLOCKLIST + parenthetical check
   - "#Overheidsinkooop" typo (3 o's) → fixed
   - "gemeente Druten" lowercase from universal_search → `fix_entity()` capitalizer
   - "In 0 jaar" when years_with_data=0 → `_in_period()` helper returns empty string
   - "Anoniem_IenW_704" anonymized entries → blocklist check for "Anoniem*"
   - "één inkoopcontracten" grammar → `_contracts()` and `_inkoopcontracten()` singular/plural helpers
   - Double spaces in entity names from source data → regex whitespace cleanup
