## Work Completed

1. **OG image (opengraph-image.tsx)** — Dynamic 1200x630 PNG via Next.js ImageResponse (edge runtime). Multiple iterations: v1 (stats bar, rejected), v2 (raster logo centered, too big), v3 (smaller logo, still blurry), v4 (all-vector text, no logo), v5 (white icon, lost detail), v6 (logo-white.png at 500px), v7 (IBM Plex Sans Bold font loaded), v8 (full font from fontsource CDN, 270 glyphs), final (left-aligned, 550px logo, 54px bold pink tagline with -0.02em letter spacing). Font bundled as local TTF (IBMPlexSans-Bold.ttf) — woff2→TTF conversion via fonttools.

2. **Sitemap (sitemap.ts)** — 6 public pages: home (priority 1), over (0.8), dataoverzicht (0.7), versiegeschiedenis (0.5), voorwaarden (0.3), privacybeleid (0.3). Note: dataoverzicht and versiegeschiedenis are behind auth intentionally — sitemap lists them for SEO value but users need login.

3. **Robots.ts update** — Added disallows: /team/, /profiel, /s/. Added sitemap reference to rijksuitgaven.nl/sitemap.xml.

4. **Layout.tsx SEO metadata** — Updated title: "Rijksuitgaven.nl | Snel inzicht voor krachtige analyses". Updated description. Added twitter card (summary_large_image). Added og:site_name. Added JSON-LD Organization schema (name, url, description, logo, contactPoint, sameAs for X/LinkedIn/Bluesky).

5. **Middleware fix** — Added `opengraph-image` to matcher exclusion in middleware.ts. Route was redirecting to /login because it wasn't in the public paths list. robots.txt and sitemap.xml were already excluded.

6. **Logo replacement** — Replaced `public/logo-white.png` with proper hi-res source from iCloud (same 1900x540 dimensions, different file — correct source asset).

7. **SPF tightening analysis** — Current SPF: `v=spf1 mx a include:spfmailpod.zxcs.nl include:rrpproxyspf.zxcs.nl include:send.resend.dev -all`. Planned new: `v=spf1 mx include:spfmailpod.zxcs.nl include:send.resend.dev -all` (remove `a` and `rrpproxyspf.zxcs.nl`). Deferred to tomorrow — user still sends from Vimexx mail, so keeping `mx` and `spfmailpod`.
