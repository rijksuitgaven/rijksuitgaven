## Work Completed

1. **Autocomplete field_matches prefix matching fix (V2.4.3)** — Field matches (regelingen, begrotingsnamen, artikelen) in the autocomplete dropdown now appear while the user is still typing partial words. Previously required exact word boundary match ("veehouderij" showed regelingen, but "veeho" didn't). Now uses exact+prefix fallback matching the same pattern as the primary field (ontvangers). Exact word boundary matches prioritized, substring matches fill remaining slots.

2. **Railway backend public networking removed** — Removed the public domain `rijksuitgaven-api-production-3448.up.railway.app` from the backend service. Frontend already uses internal URL `http://rijksuitgaven-api.railway.internal:8080`. Security improvement — backend now only accessible within Railway private network.

3. **Local dev env updated** — Changed `BACKEND_API_URL` in `app/.env.local` from production public URL to `http://localhost:8000`. Required because public networking was removed. Local dev now requires running backend locally.

4. **Bug investigation process** — Full end-to-end trace of autocomplete pipeline: Typesense queries → backend processing → BFF proxy → frontend rendering. Confirmed via production Network tab that the API returns field_matches correctly but `is_word_boundary_match` filter was too strict for partial words.

### Session 2

5. **Roadmap patches on /team/roadmap** — Parser now detects `#### Patches` headers in VERSIONING.md and parses the markdown table rows. Added `Patch` interface (id, date, changes) and `patches[]` to `Version`. UI shows collapsible "X patches" toggle under each version's features, collapsed by default.

6. **Feedback screenshot + annotation tool** — Complete redesign of the feedback widget's capture mechanism. Replaced element picker ("Markeer op de pagina") with viewport screenshot + full-screen annotation canvas. 4 drawing tools: pen (freehand), rectangle, arrow, text. All drawn in brand pink (#E62D75). Text tool places floating input at click position, commits on Enter. Undo (button + Ctrl+Z), Escape to cancel. Annotation canvas is a separate component (`annotation-canvas.tsx`, ~280 lines).

7. **html2canvas → html2canvas-pro** — Switched screenshot library from `html2canvas` (1.4.1) to `html2canvas-pro` which supports Tailwind v4's `oklab`/`oklch` color functions. html2canvas crashed on capture due to unsupported color parsing. Drop-in replacement, same API.

8. **Box-shadow stripping during capture** — html2canvas renders CSS box-shadows as thick borders (known artifact). Fix: inject `* { box-shadow: none !important }` stylesheet before capture, remove after. Clean screenshots without shadow artifacts.

9. **Remove pin limit (MAX_PINNED_ROWS)** — Removed the arbitrary 5-row pin limit. Pin button now always works. No technical reason for the limit — pinned rows are just a Map in memory. Users self-limit by screen space.
