## Summary

Admin leden page improvements: added contract_end_date column for legacy WordPress contract tracking (migration 075), fixed long-standing engagement badge bug (wrong key lookup: subscription.id vs person_id), removed Engagement column (misleading — showed email engagement, not platform usage), compacted table layout to eliminate horizontal scroll.

## Work Completed

1. **contract_end_date on leden page** — Added nullable DATE column to subscriptions table for tracking legacy WordPress contract expiry dates. Sortable "V1 Contract" column in table, date picker in edit modal ("Contractdatum (oud)"). Migration 075 executed on production. Temporary field — can be dropped when all renewals complete.

2. **Engagement badge bug fix** — Discovered and fixed a bug where engagement badges always showed "—" for all members. Root cause: engagement map was keyed by `person_id` (from `get_engagement_scores` RPC) but looked up by `member.id` (subscription id). Added `person_id` to Member interface, fixed all lookups including sort comparison.

3. **Engagement column removed** — After fixing the badges, engagement data was misleading: members with status "Uitgenodigd" (never logged in) showed "Actief" because the score was based on email opens/clicks, not platform usage. Removed column for now. Data keeps collecting in background. Statistieken page has better scoring (Power/Regulier/Casual based on actual usage).

4. **Compact leden table** — Reduced cell padding (px-4 to px-2), font size (text-sm to text-xs) to fit all 8 columns without horizontal scroll: Naam, Organisatie, E-mail, Plan, Status, Laatst actief, Einddatum, V1 Contract.

5. **Email bounce investigation (marit@nieuwrechts.nl)** — Persistent bounce after clearing bounced_at flag. Resend team analysis revealed: (a) not a suppression list issue, (b) Resend detail shows Sent > Delivered > Bounced at same timestamp, (c) "Generic Temporary Delivery Failure" from Google Workspace, (d) MX records valid (Google). Conclusion: Google Workspace issue on recipient side, nothing to fix in our code.
