## Work Completed

1. **Rule 7: Branch Discipline (No Feature Left Behind)** — New Golden Rule codified in CLAUDE.md from 3 incidents (Feb 23, Feb 24, Mar 2). 7-day max branch lifetime, daily sync with main, 6-point Pre-Merge Gate, branch state table in SESSION-CONTEXT.md, stale branches are blockers.

2. **V2.4 merge: UX-039 + UX-041** — Merged `feature/ux-039-041` (34 real commits, 5 weeks old) into main. Pre-Merge Gate 6/6: main merged into branch, 1 CSS conflict resolved (both tooltip-right and trend-colored kept), `npm run build` zero errors, user confirmed manual test, feature inventory complete, zero regression. Branch deleted.

3. **Fix: URL cols pollution from localStorage** — When navigating to a module, localStorage column preferences were encoded into the URL (`?cols=artikel,begrotingsnaam`) even though the user didn't change columns. Added `userChangedCols` ref — URL only shows `?cols=` when user actively changes columns in the session or URL originally had them.

4. **Fix: setState-during-render on expand** — `handleExpandedChange` called `router.replace()` synchronously during TanStack Table's render cycle, causing "Cannot update Router while rendering DataTable" error. Fixed with `setTimeout(0)` to defer URL update out of render phase.

5. **Fix: expand URL tracking stale row** — When multiple rows were pinned+expanded, `onExpandedChange` picked `expandedKeys[0]` which was the first pinned row, not the most recently expanded. Fixed by diffing `prev` vs `next` expanded state to find the newly expanded key.

6. **Brainstorm: cross-module pin persistence** — Analyzed 3 levels: session-only (current), localStorage persistence (complex, bug-prone), comparison workspace (proper solution). Team consensus: Level 1 now, Level 3 (Vergelijkpagina) for V2.5. Added safety check `(table.getTopRows() ?? [])` to prevent crash when pinned row IDs don't match data.

7. **V2.5 updated** — Added "Vergelijkpagina: dedicated `/vergelijk` page for cross-module comparison" to VERSIONING.md V2.5.

8. **V2.4 release notes** — Added V2.4 entry to `/versiegeschiedenis` page (pin + bookmark descriptions, benefit-first formal Dutch) and release banner in `release-notes.ts`.

9. **Help popover: pin + bookmark tips** — Added two new tips to the ℹ️ info popover in data-table: pin (Pin icon) and bookmark (bookmark icon), as separate items.

10. **Support page: Vergelijken + Bookmark** — Added "Vergelijken" and "Bookmark uw weergave" sections to `/support` under "Aan de slag".

11. **Publieke deellinks roadmap** — Added tokenized read-only URL sharing for non-subscribers to V2.5 roadmap in VERSIONING.md.

12. **V2.4 shipped to production** — Pushed 147 commits to production. Railway auto-deploy triggered.

13. **UX-043: Specific recipient picker** — New "Specifieke ontvangers" toggle below segment checkboxes on /team/mail. When enabled, fetches people matching selected segments and shows a searchable list with individual checkboxes. Selected person IDs flow through save draft, send, and draft restore. Send button count updates to show selected count. New API endpoint `GET /api/v1/team/mail/people?segments=...`.

14. **Bug fix: draft conditionGroups restore** — `handleEditDraft` was not restoring `conditionGroups` from `campaign.conditions` when editing a draft. Now restores conditions and shows the condition builder automatically.

15. **SQL migration 074** — `ALTER TABLE campaigns ADD COLUMN person_ids JSONB DEFAULT NULL` for specific recipient targeting.
