3. **Fix: URL cols pollution from localStorage** — When navigating to a module, localStorage column preferences were encoded into the URL (`?cols=artikel,begrotingsnaam`) even though the user didn't change columns. Added `userChangedCols` ref — URL only shows `?cols=` when user actively changes columns in the session or URL originally had them.

4. **Fix: setState-during-render on expand** — `handleExpandedChange` called `router.replace()` synchronously during TanStack Table's render cycle, causing "Cannot update Router while rendering DataTable" error. Fixed with `setTimeout(0)` to defer URL update out of render phase.

5. **Fix: expand URL tracking stale row** — When multiple rows were pinned+expanded, `onExpandedChange` picked `expandedKeys[0]` which was the first pinned row, not the most recently expanded. Fixed by diffing `prev` vs `next` expanded state to find the newly expanded key.

6. **Brainstorm: cross-module pin persistence** — Analyzed 3 levels: session-only (current), localStorage persistence (complex, bug-prone), comparison workspace (proper solution). Team consensus: Level 1 now, Level 3 (Vergelijkpagina) for V2.5. Added safety check `(table.getTopRows() ?? [])` to prevent crash when pinned row IDs don't match data.

Broader English detection: 20 marker words (was 7).

24. **Security audit: 15 findings resolved** — Full codebase audit with 6-person virtual team. Found and fixed:
    - **C1:** x-forwarded-host injection → ALLOWED_HOSTS whitelist in middleware.ts
    - **C2:** No CSRF protection → `csrfCheck()` requiring X-Requested-With header, 20 routes + 11 frontend files
    - **C3:** Backend publicly accessible → Railway private networking enabled (internal-only)
    - **C4:** No backend rate limiting → Token bucket middleware (10/min public, 120/min authenticated)
    - **H1:** CSP allows unsafe-eval → Removed from next.config.ts
    - **H2:** getSession() trusts unverified JWT → Replaced with getUser() (server-verified)
    - **H3:** listUsers unbounded pagination → Direct GoTrue API call
    - **H4:** Contact form no rate limit → In-memory rate limit + honeypot field
    - **H5:** NEXT_PUBLIC_TYPESENSE exposed → Removed from .env.local (never on Railway)
    - **H6:** xlsx CVE → Replaced with ExcelJS
    - **M1:** Python deps outdated → fastapi 0.135.1, python-multipart 0.0.22
    - **M2:** _la cookie not secure → Added secure: true flag
    - **L1:** Console.error leaks internal details → Sanitized error output
    - **RLS audit:** 23 tables verified — all properly secured
    - **Dependency audit:** npm audit 0 vulns, pip audit 0 vulns

25. **Railway private networking** — Backend service now only accessible via `rijksuitgaven-api.railway.internal:8080`. Public networking removed. Frontend BFF is the only path to backend.

26. **Bug fix: "slaap" compound word search** — 4-commit fix. Root cause 1: `is_word_boundary_match()` used `\b` regex which rejects Dutch compound words — skipped for primary field (a0c3c41). Root cause 2: case mismatch between Typesense raw values and title-cased aggregated view (`UPPER(LEFT(MIN(ontvanger),1))`) — fixed with `LOWER()` in WHERE clause (ec292de). Root cause 3: Next.js caching stale responses — added `cache: 'no-store'` to BFF fetch + `force-dynamic` on all 8 module route files (8493b5e, fedbc61). Root cause 4: Chrome disk cache serving old responses — upgraded Cache-Control to `no-store` and added `cache: 'no-store'` to all client-side fetches (73fdb21).

27. **Browser cache prevention** — Upgraded `Cache-Control` from `private, no-cache` to `private, no-cache, no-store, must-revalidate` on all BFF proxy responses. Added `cache: 'no-store'` to all 5 client-side `fetch()` calls (modules list, module data, detail data, cascading filters, module stats). Prevents browser disk cache from ever serving stale API responses.
