## Work Completed

1. **Rule 7: Branch Discipline (No Feature Left Behind)** — New Golden Rule codified in CLAUDE.md from 3 incidents (Feb 23, Feb 24, Mar 2). 7-day max branch lifetime, daily sync with main, 6-point Pre-Merge Gate, branch state table in SESSION-CONTEXT.md, stale branches are blockers.

2. **V2.4 merge: UX-039 + UX-041** — Merged `feature/ux-039-041` (34 real commits, 5 weeks old) into main. Pre-Merge Gate 6/6: main merged into branch, 1 CSS conflict resolved (both tooltip-right and trend-colored kept), `npm run build` zero errors, user confirmed manual test, feature inventory complete, zero regression. Branch deleted.

3. **Fix: URL cols pollution from localStorage** — When navigating to a module, localStorage column preferences were encoded into the URL (`?cols=artikel,begrotingsnaam`) even though the user didn't change columns. Added `userChangedCols` ref — URL only shows `?cols=` when user actively changes columns in the session or URL originally had them.

4. **Fix: setState-during-render on expand** — `handleExpandedChange` called `router.replace()` synchronously during TanStack Table's render cycle, causing "Cannot update Router while rendering DataTable" error. Fixed with `setTimeout(0)` to defer URL update out of render phase.

5. **Fix: expand URL tracking stale row** — When multiple rows were pinned+expanded, `onExpandedChange` picked `expandedKeys[0]` which was the first pinned row, not the most recently expanded. Fixed by diffing `prev` vs `next` expanded state to find the newly expanded key.

6. **Brainstorm: cross-module pin persistence** — Analyzed 3 levels: session-only (current), localStorage persistence (complex, bug-prone), comparison workspace (proper solution). Team consensus: Level 1 now, Level 3 (Vergelijkpagina) for V2.5. Added safety check `(table.getTopRows() ?? [])` to prevent crash when pinned row IDs don't match data.

7. **V2.5 updated** — Added "Vergelijkpagina: dedicated `/vergelijk` page for cross-module comparison" to VERSIONING.md V2.5.

8. **V2.4 release notes** — Added V2.4 entry to `/versiegeschiedenis` page (pin + bookmark descriptions, benefit-first formal Dutch) and release banner in `release-notes.ts`.

9. **Help popover: pin + bookmark tips** — Added two new tips to the ℹ️ info popover in data-table: pin (Pin icon) and bookmark (bookmark icon), as separate items.

10. **Support page: Vergelijken + Bookmark** — Added "Vergelijken" and "Bookmark uw weergave" sections to `/support` under "Aan de slag".

11. **Publieke deellinks roadmap** — Added tokenized read-only URL sharing for non-subscribers to V2.5 roadmap in VERSIONING.md.

12. **V2.4 shipped to production** — Pushed 147 commits to production. Railway auto-deploy triggered.

13. **UX-043: Specific recipient picker** — New "Specifieke ontvangers" toggle below segment checkboxes on /team/mail. When enabled, fetches people matching selected segments and shows a searchable list with individual checkboxes. Selected person IDs flow through save draft, send, and draft restore. Send button count updates to show selected count. New API endpoint `GET /api/v1/team/mail/people?segments=...`.

14. **Bug fix: draft conditionGroups restore** — `handleEditDraft` was not restoring `conditionGroups` from `campaign.conditions` when editing a draft. Now restores conditions and shows the condition builder automatically.

15. **SQL migration 074** — `ALTER TABLE campaigns ADD COLUMN person_ids JSONB DEFAULT NULL` for specific recipient targeting.

16. **UX-044: Uitgeschreven filter** — Added "uitgeschreven" tracking to /team/leden and /team/contacten. Clickable count in stats bar filters the table. Visual indicator (strikethrough email icon) on rows where `unsubscribed_at IS NOT NULL`. Orthogonal to membership status — someone can be "Actief" but unsubscribed from emails.

17. **Social pipeline rewrite: requirements gathering** — Two rounds of Q&A (31 questions total) with virtual senior team (Adversarial Strategist, Dutch copywriter, data engineer, content strategist). Covered: audience (journalists, policy wonks, general public), tone (factual only, no opinion), templates (C7-style proven patterns), volume (500 posts, 3/day), scoring (rule-based only, skip Claude API), hashtags (only #Rijksuitgaven), platform (same text across X/LinkedIn/Bluesky via Buffer CSV).

18. **Social pipeline rewrite: extract_facts.py** — Complete rewrite from 19 aggregated view queries to 7 source table queries. Each extracts per-(recipient, year, descriptor) rows directly from source tables. Module-specific thresholds: €100K instrumenten (×1000), €1M apparaat (×1000), staffel≥7 for inkoop/COA, €25K for provincie/gemeente/publiek. Years 2022-2024 only. Publiek fix: COALESCE(regeling, omschrijving) for ZonMW/NWO empty descriptors. 19 old fact CSVs auto-cleaned up.

19. **Social pipeline rewrite: generate_posts.py** — Complete rewrite with 3 template sets: Standard (14 universal + module-specific extras for instrumenten/provincie/gemeente/publiek), Apparaat (8 internal cost templates), Staffel (6 inkoop + 6 COA bracket-based). Filters: expanded entity blocklist, tautology detection (bidirectional substring), clean_descriptor (budget code prefix stripping, address filter, English research title filter, "Boekjaar YYYY" skip). Selection: round-robin across modules, max 3 per recipient, 500 total. Single output: `posts/buffer-ready.csv`.

20. **Social pipeline: filter iterations** — Fixed 8 data quality issues discovered during generation: (1) "Particulier" as entity → blocklist, (2) address-like descriptors → regex filter, (3) budget code prefixes ("1 Ruimtelijke Ontwikkeling - 2.11 ...") → hierarchical stripping, (4) "1542 natuurlijke personen" entities → regex blocklist, (5) English research titles from ZonMW/NWO → function word marker filter, (6) descriptor=entity tautology ("BNI-afdrachten") → is_tautological check, (7) "Geanonimiseerde ontvanger(s)" → extended anonymized detection, (8) empty publiek descriptors (395/500) → COALESCE fix.

21. **Social: template curation** — User-reviewed all 48 templates one by one. Cut 16: bad Dutch (descriptors "keerde uit", "leverde op"), filler ("Wist u dat", "Kort:", "Samengevat:", "Resultaat van"), and weak structures ("Via ... ging ... naar", "Voor ... betekende"). 32 templates remain across 8 sets.

22. **Social: contextual hashtags** — Replaced single #Rijksuitgaven with 2-3 contextual hashtags per post. Added `build_hashtags()`, `make_entity_tag()`, `make_ministry_tag()`, `make_kostensoort_tag()`, `make_category_tag()` functions. Ministry map (13 ministries → short tags like #Defensie, #BZK, #VWS). Category map (12 inkoop categories → tags like #ICT, #Advies). Entity tag rules: strip B.V./N.V., skip Stichting/Gemeente prefix, max 2 words, max 25 chars. Geography tags from provincie/gemeente names.

23. **Social: 50-char max everywhere** — Lowered max_len from 80→50 in `clean_descriptor()`. Also applied 50-char limit to: kostensoort, begrotingsnaam (apparaat), categorie (inkoop), regeling (COA), entity names (is_blocked). Skip entirely instead of truncating — catches source-truncated descriptors like "kindermi" (from gemeente source table). Also skip descriptors ending with `-` or `…`. Broader English detection: 20 marker words (was 7).
