## Work Completed

1. **Social template updates** — 4 changes across 3 template sets:
   - Provincie: template 9 "via" → "voor"; added template 10a ("maakte ... over aan")
   - Gemeente: template 11 "keerde uit" → "betaalde"; added template 13a ("maakte ... over aan")
   - Inkoop: added template 26a ("De Rijksoverheid gunde in {jaar} opdrachten aan {leverancier} met een waarde van {bracket}")
   - COA: all 5 templates rewritten with `{jaar}`; fixed template 27 "tussen {bracket}" double-prefix bug (user had "tussen" before bracket which already contains "tussen")

2. **extract_facts.py: jaar added to COA and inkoop** — COA query now groups by `normalize_recipient(ontvanger), jaar` (was just ontvanger). Inkoop query now groups by `normalize_recipient(leverancier), staffel, jaar` (was just leverancier, staffel). Both queries filtered to 2022-2024. CSV headers updated.

3. **Verification system for social posts** — Every template variable used in a post is verified against the source data. Pipeline: (1) choose template, (2) extract `{field_name}` placeholders via `get_template_fields()`, (3) format text with values from source CSV, (4) verify each used value appears in the final text, (5) if ANY field fails → reject post entirely. CSV output: 15 variable columns with "value [Checked & Verified]" when used, empty when not applicable to that module.

4. **Template-aware field verification** — Initial implementation verified ALL fields in the dict, causing 298 instrumenten posts to fail (standard templates don't use `{type}`, but `type` was in the fields dict). Fixed by only verifying fields that the chosen template actually uses. Uses regex extraction of `{...}` placeholders from the template string.

5. **New CSV output format** — `buffer-ready.csv` now has 18 columns: text, module, chars, + 15 template variable columns (ontvanger, bedrag, jaar, descriptor, type, provincie, gemeente, source, begrotingsnaam, kostensoort, leverancier, bracket, categorie, regeling, hashtags). Each variable column shows "value [Checked & Verified]" when used in that post, empty when not applicable.
